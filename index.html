Using other people's code

We can use other people's code to effectively save time and effort, which will let us add powerful functionality to our code. Object-oriented programming (OOP) is a programming language model organized around objects rather than "actions" and data rather than logic.

An Object is an instance of a class. An instance is an object of a class created at run-time. The terms object and instance are normally used synonymously.

Once an object is identified, it is generalized as a class of objects which defines the kind of data it contains and any logic sequences that can manipulate it. Each distinct logic sequence is known as a method.

A Method is a function, but it is a special kind of function which belongs to a class, in other words, it is defined in a class, and works on the instance and class data of this class. The special thing about Methods is that the object is passed as the first argument of the function. Methods can only be called through instances of the BaseClass or a DerivedClass with the BaseClass defined in the scope of the containing the derived class definition. For example:

class DerivedClass(BaseClass):

The class name followed by a dot and the method name. like in the inheritance project we called miley_cyrus.show_info() we could do this because we defined this method in the Parent class billy_cyrus.

Inheritance, Abstraction and how it relates

Inheritance We use inheritance for resolving attribute references: if a requested attribute is not found in the DerivedClass, the search proceeds to look in the BaseClass. This rule is applied recursively if the base class itself is derived from some other class. For example we can create a class called Video's which will be the BaseClass then we can have two DerivedClasses the class movies, that we have already written, and then create a new class called Tv-Shows. Then the DerivedClasses will get some attributes from the BaseClass if not already defined within that instance.

Abstraction is the process of taking away or removing chracteristics from something in order to reduce it to a set of essential characteristics. Throught the process of abstraction, a programmer hides all but the relevant data about an object in order to reduce complexity and increse efficiency.
